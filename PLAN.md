## 既知の課題

### Medium インポーター

- **画像が外部URL依存**: インポーター（`.github/workflows/medium-import.yml`）は画像をダウンロードしない。Medium の CDN URL がそのままコンテンツに残るため、Medium 側で画像が削除・移動されると壊れる。
- **重複チェックなし**: 同じ記事が複数回インポートされる可能性がある。`scripts/duplicate_post_cleaner.rb`（85% 類似度）で事後対処は可能だが、インポート時点で防ぐ仕組みはない。
- **過去7日間のみ対象**: インポートは毎週日曜の直近7日間の記事を対象にするため、ワークフローがスキップされた週の記事は取得されない。

### ディレクトリ構造

- `content/posts/draft_-----------90caff9be54d/` が存在する。通常の `YYYYMM/UUID` 命名規則と異なる。意図的なドラフトなのか、インポートエラーの残骸なのか不明。
- `scripts/` に `.bak` ファイルや一度しか使われないマイグレーションスクリプトが残っている。

### i18n

- `i18n/` ディレクトリが空。多言語対応の構造だけ存在して未実装。当面日本語のみなので放置中。

### ログの蓄積

- `scripts/logs/medium_import_summary.json` に毎週のインポートログが追記される。定期的なクリーンアップの仕組みはない。

## 設計メモ（決定済み事項）

### Firebase API キーの公開

`static/js/firebase-config.js` に API キーが含まれており git 管理される。これは意図的な設計で、Firebase の公開キーはフロントエンドに埋め込まれることが前提。セキュリティはFirebase Security Rules で担保する。

### タイムゾーン対応の日付表示

Hugo の `DateFormat: "2006-01-02"` で日付のみ表示。時刻を表示すると JST/UTC の時差でずれるため省略している。

### slug = UUID

Medium の記事 UUID を slug として使用（`/:year:month:slug/` → `/202506xxxxxxxx/`）。可読性より安定性を優先し、Medium の URL との紐付けを維持している。
